📌 일정 등록 및 표시 관련 기능 요구사항 정리

✅ 1. 일정 유형별 입력 필드 동작
* 연차 / 반차 선택 시
    * 시작일시, 종료일시 필드는 숨김 처리
    * 대신 사용일 필드 표시 (날짜만 선택 가능)
    * 반차 선택 시, 추가로 오전 / 오후 선택 가능하도록 유형 필드 노출
* 회의 / 스크럼 선택 시 참가자 선택 영역
    * 전사 일정: 전체 인원 자동 선택
    * 기획팀 일정: 기획팀 인원 자동 선택
    * 디자인팀 일정: 디자인팀 인원 자동 선택
    * 프론트엔드팀 일정: 프론트엔드 인원 자동 선택
    * 백엔드팀 일정: 백엔드 인원 자동 선택
    * "하루종일" 체크박스는 제거
    * 시작일시, 종료일시는 30분 단위로 시간 선택 가능하도록 설정
* 회의 / 스크럼 선택 시 ui 개선 요구사항
* 참가자 목록을 아래와 같이 팀별로 구분된 섹션으로 나누어 표시
    * 기획팀
    * 디자인팀
    * 프론트엔드팀
    * 백엔드팀
* 

✅ 2. 캘린더 색상 및 필터 기능
* 일정 유형별로 고유한 색상을 적용해야 함 (예: 전사일정=주황, 영업팀=보라 등)
* 색상은 다음에 적용되어야 함:
    * 일정 목록
    * 캘린더 셀의 배경색 (예시: '휴가' 일정은 연한 분홍색 배경)
* Schedule 페이지에서는 아래 항목 기준으로 일정 필터링 기능 제공 (다중 선택 가능):
    * 전사 일정
    * 기획팀 일정
    * 디자인팀 일정
    * 프론트엔드팀 일정
    * 백엔드팀 일정
