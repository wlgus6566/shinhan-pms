# AI 에이전트 워크플로우

## 전체 프로세스

```
사용자 요구사항
    ↓
PM (요구사항 분석, 우선순위 결정)
    ↓
Planner (기획서 작성)
    ↓
PM 검수 → 승인
    ↓
Wireframe (HTML 와이어프레임 작성)
    ↓
PM 검수 → 승인
    ↓
Modeler (DB 스키마 설계)
    ↓
PM 검수 → 승인 → 마이그레이션
    ↓
Developer (백엔드 개발 - TDD)
    ↓
PM 검수 → 승인
    ↓
Developer (프론트엔드 개발)
    ↓
PM 검수 → 최종 승인
    ↓
기능 완료
```

## 단계별 상세

### 1. PM (프로젝트 관리자)
- 요구사항 분석
- 우선순위 결정
- 각 단계별 검수 및 승인
- todo.md 관리

### 2. Planner (기획자)
- 사용자 스토리 작성
- 기능 요구사항 정의
- API 경로 설계
- 화면 목록 및 흐름 정의
- **금지**: DB 테이블 상세 설계, API 상세 JSON 작성, HTML 작성

### 3. Wireframe (와이어프레임 제작자)
- 기획서 기반 HTML 와이어프레임 작성
- 화면 간 링크 연결
- 기능 표현에 집중
- **금지**: 와이어프레임 임의 수정

### 4. Modeler (DB 모델러)
- Prisma 스키마 작성
- DDL/DML 작성
- 인덱스 및 외래 키 설계
- **필수**: 공통 컬럼 포함

### 5. Developer (풀스택 개발자)
- 백엔드: TDD 기반 개발 (테스트 → 구현 → 리팩토링)
- 프론트엔드: 와이어프레임 기반 UI 구현
- **금지**: 와이어프레임 임의 수정

## 순차 진행 원칙

- Planner → Wireframe → Modeler → Developer 순서 엄수
- 이전 단계 미승인 시 다음 단계 진행 금지
- PM 승인 없이 마이그레이션 금지
