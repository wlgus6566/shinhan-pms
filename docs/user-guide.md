## 1. 시스템 소개

**Emotion PMS**는 프로젝트와 업무를 효율적으로 관리하기 위한 웹 기반 프로젝트 관리 시스템입니다.

### 주요 기능

| 기능 | 설명 |
|------|------|
| 프로젝트 관리 | 프로젝트 생성, 팀원 배정, 진행 현황 관리 |
| 업무 관리 | 프로젝트별 업무 등록, 담당자 배정, 상태 추적 |
| 업무일지 | 일일 작업 내용 기록, 진행률 및 작업시간 관리 |
| 일정 관리 | 팀 일정 공유, 반복 일정, 참가자 관리 |
| 리포트 | 프로젝트 생산성 분석, Excel 보고서 내보내기 |
| 멤버 관리 | 사용자 계정 생성 및 권한 관리 (관리자 전용) |

### 접속 정보

- **웹 주소**: 배포 환경에 따라 다름 (개발: `http://localhost:3001`)
- **지원 브라우저**: Chrome, Edge, Safari, Firefox (최신 버전 권장)
- **모바일 지원**: 반응형 디자인으로 모바일 브라우저에서도 이용 가능

## 2. 로그인

### 로그인 방법

1. 웹 브라우저에서 Emotion PMS 주소에 접속합니다.
2. **이메일**과 **비밀번호**를 입력합니다.
3. **로그인** 버튼을 클릭합니다.
4. 인증 성공 시 대시보드 화면으로 이동합니다.

### 로그아웃

- 좌측 사이드바 하단의 사용자 프로필 영역에서 **로그아웃 아이콘**을 클릭합니다.

### 참고

- 계정은 관리자(SUPER_ADMIN)가 생성합니다. 직접 회원가입은 불가합니다.
- 비밀번호 분실 시 관리자에게 문의하세요.

## 3. 화면 구성

### 사이드바 메뉴

로그인 후 좌측에 네비게이션 사이드바가 표시됩니다.

```
Emotion PMS (로고)
├── 대시보드          ← 업무 현황 요약
├── 프로젝트 관리      ← 프로젝트 목록 및 관리
├── 업무 관리          ← 드롭다운 (내 프로젝트별 업무)
│   ├── 프로젝트 A
│   ├── 프로젝트 B
│   └── ...
├── 업무일지          ← 일일 작업 기록
├── 일정 관리          ← 캘린더 기반 일정
├── 프로젝트 리포트    ← 통계 및 분석
│
├── [관리자] ──────────
│   └── 멤버 관리      ← 사용자 계정 관리
│
├── 진행중인 내 프로젝트: N개
│
└── 사용자 프로필 / 로그아웃
```

### 사이드바 조작

- **접기/펼치기**: 사이드바 우측 가장자리의 화살표 버튼(`<` / `>`)으로 토글
- **접힌 상태**: 아이콘만 표시되며, 마우스를 올리면 메뉴명이 툴팁으로 표시
- **모바일**: 좌측 상단 햄버거 메뉴로 사이드바 열기/닫기

## 4. 대시보드

> 경로: `/dashboard`

로그인 후 가장 먼저 보이는 화면으로, 업무 현황을 한눈에 파악할 수 있습니다.

### 구성 요소

#### 프로필 카드
- 이름, 직급, 부서 정보 표시
- 오늘의 날짜와 날씨 정보

#### 핵심 통계 (상단 카드)

| 항목 | 설명 |
|------|------|
| 진행중 프로젝트 | 현재 활성 상태의 프로젝트 수 |
| 내 업무 - 진행중 | 나에게 배정된 진행중 업무 수 |
| 이번 주 작업시간 | 금주 업무일지에 기록된 총 작업시간 |
| 중요 업무 | 난이도 HIGH인 업무 수 |

#### 오늘의 통계
- **만료 예정 업무**: 종료일이 임박한 업무
- **예정된 일정**: 오늘 예정된 일정

#### 바로가기 카드
- 업무 관리 (PM인 프로젝트 최대 3개로 바로 이동)
- 프로젝트 관리
- 일정 확인

#### 팀 활동 타임라인
- 최근 5건의 팀 활동 (업무일지 작성, 업무 생성 등)

#### 다가오는 일정
- 예정된 일정 목록

## 5. 프로젝트 관리

> 경로: `/projects`

### 5.1 프로젝트 목록 조회

프로젝트 관리 메뉴에 진입하면 프로젝트 목록이 표시됩니다.

**필터 및 검색**:
- **검색**: 프로젝트명으로 검색
- **상태 필터**: 진행중 / 완료 / 보류

### 5.2 프로젝트 생성

> 권한: PM, SUPER_ADMIN

1. 프로젝트 목록 화면에서 **새 프로젝트** 버튼 클릭
2. 아래 항목을 입력:

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 프로젝트명 | O | 프로젝트 이름 |
| 클라이언트 | - | 발주처/고객사 |
| 프로젝트 타입 | - | 프로젝트 유형 |
| 설명 | - | 프로젝트 상세 설명 |
| 시작일 | - | 프로젝트 시작 예정일 |
| 종료일 | - | 프로젝트 종료 예정일 |
| 상태 | O | 진행중(ACTIVE) / 완료(COMPLETED) / 보류(SUSPENDED) |

3. **저장** 버튼 클릭

### 5.3 프로젝트 상세

프로젝트를 클릭하면 상세 페이지로 이동합니다. 탭 구조로 다양한 정보를 확인할 수 있습니다.

| 탭 | 내용 |
|----|------|
| **개요** | 프로젝트 기본 정보, 팀원 목록 |
| **업무 관리** | 프로젝트 내 전체 업무 목록 |
| **일지 현황** | 팀원들의 업무일지 현황 |
| **일정** | 프로젝트 관련 일정 |
| **리포트** | 주간/월간 보고서 다운로드 |

### 5.4 팀원 관리

프로젝트 상세 > 개요 탭에서 팀원을 관리합니다.

**팀원 추가**:
1. **팀원 추가** 버튼 클릭
2. 추가할 멤버 선택
3. 역할(PM/PL/PA/Member) 및 업무 분야(기획/디자인/프론트엔드/백엔드) 지정
4. 저장

**팀원 역할 변경**: 팀원 목록에서 역할이나 분야를 수정 후 저장

**팀원 제거**: 팀원 목록에서 삭제 버튼 클릭

### 5.5 프로젝트 수정

> 권한: PM 또는 프로젝트 생성자

프로젝트 상세 화면에서 **편집** 버튼을 클릭하여 정보를 수정합니다.

## 6. 업무 관리

> 경로: `/tasks/{프로젝트ID}`

사이드바의 **업무 관리** 메뉴를 클릭하면 드롭다운으로 내가 참여 중인 프로젝트 목록이 표시됩니다. 원하는 프로젝트를 선택하면 해당 프로젝트의 업무 목록 페이지로 이동합니다.

### 6.1 업무 목록 조회

**필터 및 검색**:
- **검색**: 업무명으로 검색
- **상태 필터**: 대기 / 진행 / 완료 / 보류
- **난이도 필터**: 낮음 / 중간 / 높음

### 6.2 업무 생성

> 권한: PM

1. 업무 목록 화면에서 **새 업무** 버튼 클릭
2. 아래 항목을 입력:

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 업무명 | O | 업무 이름 |
| 업무유형 | O | 업무 카테고리 |
| 난이도 | O | LOW / MEDIUM / HIGH |
| 상태 | O | WAITING / IN_PROGRESS / COMPLETED / ON_HOLD |
| 설명 | - | 업무 상세 설명 |
| 시작일 | - | 업무 시작 예정일 |
| 종료일 | - | 업무 종료 예정일 |

3. **담당자 배정**: 분야별로 담당자를 지정합니다.

| 분야 | 설명 |
|------|------|
| 기획 (PLANNING) | 기획 담당자 |
| 디자인 (DESIGN) | 디자인 담당자 |
| 프론트엔드 (FRONTEND) | 프론트엔드 개발 담당자 |
| 백엔드 (BACKEND) | 백엔드 개발 담당자 |

> 각 분야마다 여러 명의 담당자를 지정할 수 있습니다.

4. **저장** 버튼 클릭

### 6.3 업무 상세

업무를 클릭하면 상세 정보를 확인할 수 있습니다.

- 업무 기본 정보 (상태, 난이도, 기간 등)
- 분야별 담당자 목록
- 해당 업무의 업무일지 히스토리

### 6.4 업무 수정/삭제

> 권한: PM

업무 상세 화면에서 **편집** 또는 **삭제** 버튼을 사용합니다.

## 7. 업무일지

> 경로: `/work-logs`

### 7.1 화면 구성

- **달력 뷰**: 월간 캘린더에 일지 작성 현황이 시각적으로 표시됩니다.
- **프로젝트 탭**: 상단 탭으로 프로젝트별 필터링이 가능합니다.

### 7.2 업무일지 작성

1. 캘린더에서 원하는 **날짜를 클릭**합니다.
2. 일지 작성 폼이 나타나면 아래 항목을 입력합니다:

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 작업일자 | O | 작업 수행일 |
| 담당 업무 | O | 드롭다운에서 나에게 배정된 업무 선택 |
| 작업 내용 | O | 수행한 작업의 상세 내용 |
| 작업 시간 | O | 해당 업무에 투입한 시간 (숫자) |
| 진행률 | - | 업무 진행 정도 (0~100%) |
| 이슈/장애 사항 | - | 작업 중 발생한 이슈 |

3. **저장** 버튼 클릭

### 7.3 편리한 기능

- **작업 내용 추천**: 이전에 작성한 내용이 자동 제안되어 빠르게 입력 가능
- **다중 일지 편집**: 같은 날짜에 여러 업무의 일지를 일괄 수정 가능
- **일지 수정/삭제**: 본인이 작성한 일지만 수정/삭제 가능

### 7.4 보고서 내보내기 (Excel)

프로젝트 상세 > 리포트 탭에서 보고서를 다운로드할 수 있습니다.

| 보고서 | 설명 |
|--------|------|
| **주간 보고서** | 선택한 주의 업무 수행 내역 |
| **월간 투입인력별 상세업무현황** | 월간 팀원별 업무 수행 상세 |
| **월간 업무별 공수투입현황** | 월간 업무별 투입 공수 현황 |

## 8. 일정 관리

> 경로: `/schedule`

### 8.1 화면 구성

- **프로젝트 탭**: 내가 참여 중인 프로젝트별로 일정 필터링
- **월간 캘린더**: 일정이 색상별로 표시됨

### 8.2 일정 생성

1. 캘린더에서 원하는 **날짜를 클릭**합니다.
2. 일정 생성 폼이 나타나면 아래 항목을 입력합니다:

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 일정명 | O | 일정 제목 |
| 일정 유형 | O | 아래 표 참고 |
| 설명 | - | 일정 상세 설명 |
| 시간 설정 | O | 종일 / 시작~종료 시간 지정 |
| 위치 | - | 장소 |
| 색상 | - | 캘린더 표시 색상 |
| 참가자 | - | 일정에 초대할 팀원 |
| 반복 설정 | - | 반복 일정인 경우 설정 |

**일정 유형**:

| 유형 | 설명 |
|------|------|
| 회의 (MEETING) | 일반 회의 |
| 스크럼 (SCRUM) | 데일리 스크럼 등 |
| 휴가 (VACATION) | 종일 휴가 |
| 반차 (HALF_DAY) | 오전/오후 반차 |
| 기타 (OTHER) | 기타 일정 |

3. **저장** 버튼 클릭

### 8.3 반복 일정

반복 일정을 설정하면 지정한 기간 동안 자동으로 일정이 반복 생성됩니다.

| 반복 유형 | 설명 |
|-----------|------|
| 매일 (DAILY) | 매일 반복 |
| 매주 (WEEKLY) | 특정 요일마다 반복 |
| 매월 (MONTHLY) | 매월 같은 날 반복 |

### 8.4 참가자 관리

- 일정 생성 시 팀원을 **참가자로 초대**할 수 있습니다.
- 초대받은 참가자는 **수락(ACCEPTED) / 거절(DECLINED)** 응답이 가능합니다.
- 미응답 시 **대기(PENDING)** 상태로 표시됩니다.

### 8.5 일정 수정/삭제

- 본인이 생성한 일정만 수정/삭제할 수 있습니다.

## 9. 프로젝트 리포트

> 경로: `/analytics`

### 9.1 화면 구성

상단에서 **프로젝트**와 **조회 월**을 선택하면 해당 기간의 분석 데이터가 표시됩니다.

### 9.2 통계 항목

#### 생산성 통계 카드

| 항목 | 설명 |
|------|------|
| 총 작업 시간 | 선택 기간의 총 투입 시간 |
| 완료된 업무 | 완료 상태의 업무 수 |
| 진행중 업무 | 진행 중인 업무 수 |
| 지연된 업무 | 종료일이 지난 미완료 업무 수 |

#### 차트 및 분석

| 차트 | 설명 |
|------|------|
| **상태별 진행 건수** | 대기/진행/완료/보류별 업무 수 (파이/막대 차트) |
| **분야별 작업 시간 분포** | 기획/디자인/프론트엔드/백엔드별 작업 시간 비율 |
| **파트별 담당 업무 건수** | 각 분야별 배정된 업무 수 |
| **파트별 일일 평균 근무 시간** | 각 분야의 일 평균 작업 시간 |

### 9.3 권한별 조회 범위

| 역할 | 조회 범위 |
|------|-----------|
| 일반 사용자 (USER) | 본인 데이터만 |
| PM / SUPER_ADMIN | 프로젝트 전체 데이터 |

## 10. 멤버 관리 (관리자)

> 경로: `/users`
> 권한: PM, SUPER_ADMIN만 접근 가능

### 10.1 멤버 목록 조회

**필터 및 검색**:
- **검색**: 이름 또는 이메일로 검색
- **부서 필터**: 파트(부서)별 필터링
- **등급 필터**: 역할(SUPER_ADMIN/PM/USER)별 필터링
- **활성화 필터**: 활성/비활성 계정 필터

### 10.2 멤버 생성

> 권한: SUPER_ADMIN만 가능

1. 멤버 목록에서 **새 멤버** 버튼 클릭
2. 아래 항목을 입력:

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 이메일 | O | 로그인에 사용할 이메일 |
| 이름 | O | 사용자 이름 |
| 부서 | O | 소속 부서(파트) |
| 직급 | O | 직급 |
| 역할 | O | SUPER_ADMIN / PM / USER |

3. **저장** 버튼 클릭

> 초기 비밀번호는 시스템에서 설정됩니다.

### 10.3 멤버 정보 수정

> 권한: SUPER_ADMIN만 가능

멤버 상세에서 이름, 부서, 직급 등을 수정할 수 있습니다.

### 10.4 멤버 비활성화

> 권한: SUPER_ADMIN만 가능

- 멤버를 비활성화하면 해당 계정으로 로그인이 불가합니다.
- **본인 계정은 비활성화할 수 없습니다.**

## 13. 자주 묻는 질문 (FAQ)

### Q. 비밀번호를 잊어버렸어요.
**A.** 시스템 관리자(SUPER_ADMIN)에게 비밀번호 초기화를 요청하세요.

### Q. 프로젝트가 사이드바 업무 관리에 안 보여요.
**A.** 업무 관리 드롭다운에는 **내가 참여 중인 프로젝트**만 표시됩니다. 프로젝트 팀원으로 추가되어 있는지 PM에게 확인하세요.

### Q. 업무를 생성할 수 없어요.
**A.** 업무 생성은 **PM 역할**을 가진 사용자만 가능합니다. PM에게 업무 등록을 요청하세요.

### Q. 업무일지를 수정하고 싶어요.
**A.** 캘린더에서 해당 날짜를 클릭하면 작성한 일지를 수정할 수 있습니다. **본인이 작성한 일지만** 수정 가능합니다.

### Q. 멤버 관리 메뉴가 안 보여요.
**A.** 멤버 관리는 **PM 또는 SUPER_ADMIN** 역할에게만 표시됩니다. 권한이 필요한 경우 관리자에게 문의하세요.

### Q. 프로젝트 리포트에서 팀 전체 데이터가 안 보여요.
**A.** 일반 사용자(USER)는 본인 데이터만 조회 가능합니다. 프로젝트 전체 데이터는 **PM 이상**의 역할이 필요합니다.

### Q. 모바일에서도 사용할 수 있나요?
**A.** 네, 반응형 디자인이 적용되어 있어 모바일 브라우저에서도 이용 가능합니다. 좌측 상단의 햄버거 메뉴 아이콘으로 사이드바를 열 수 있습니다.

### Q. Excel 보고서는 어디서 다운로드하나요?
**A.** 프로젝트 상세 페이지 > **리포트** 탭에서 주간/월간 보고서를 Excel로 내보낼 수 있습니다.
