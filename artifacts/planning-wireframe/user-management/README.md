# 회원 관리 와이어프레임

## 화면 목록

1. **signup.html** - 회원가입 화면
   - 기능: 이메일, 비밀번호, 이름, 파트 입력하여 가입
   - 검증: 이메일 형식, 비밀번호 정책, 비밀번호 일치 확인
   - 이동: 로그인 화면

2. **login.html** - 로그인 화면
   - 기능: 이메일, 비밀번호로 로그인
   - 이동: 대시보드 (프로젝트 목록), 회원가입

3. **profile.html** - 프로필 조회/수정 화면
   - 기능: 본인 정보 조회 및 수정 (이름, 파트)
   - 제약: 이메일, 등급은 수정 불가
   - 이동: 비밀번호 변경

4. **password.html** - 비밀번호 변경 화면
   - 기능: 현재 비밀번호 확인 후 새 비밀번호로 변경
   - 검증: 비밀번호 정책, 비밀번호 일치 확인
   - 이동: 프로필

5. **users.html** - 사용자 목록 화면 (PM/PL 전용)
   - 기능: 전체 사용자 목록 조회, 검색, 필터
   - 필터: 파트별, 등급별, 활성/비활성
   - 이동: 사용자 상세

6. **user-detail.html** - 사용자 상세/수정 화면 (PM 전용)
   - 기능: 멤버 정보 수정 (파트, 등급, 활성 상태)
   - 제약: 본인 등급 변경 불가
   - 이동: 사용자 목록

## 화면 흐름

```
[로그인 화면] (login.html)
├─ [회원가입] → [회원가입 화면] (signup.html) → (가입 완료) → [로그인 화면]
├─ (로그인 성공) → [프로젝트 목록] (../project-management/index.html)

[헤더 메뉴] (로그인 후)
├─ [프로필] → [프로필 조회/수정] (profile.html)
│   └─ [비밀번호 변경] → [비밀번호 변경 화면] (password.html) → (저장) → [프로필]
│
└─ [사용자 관리] (PM/PL만 표시) → [사용자 목록] (users.html)
    └─ [사용자 선택] → [사용자 상세/수정] (user-detail.html) → (저장) → [사용자 목록]
```

## 권한 구분

### 비로그인 사용자
- signup.html (회원가입)
- login.html (로그인)

### 로그인 사용자 (전체)
- profile.html (본인 프로필)
- password.html (비밀번호 변경)

### PM/PL 사용자
- users.html (사용자 목록 조회)

### PM 사용자
- user-detail.html (멤버 정보 수정)

## 로컬에서 확인하기

```bash
# 간단한 HTTP 서버 실행
cd artifacts/planning-wireframe/user-management
python3 -m http.server 8000

# 브라우저에서 열기
open http://localhost:8000/login.html
```

## 주요 기능

### 회원가입 (signup.html)
- 이메일 형식 검증
- 비밀번호 정책 검증 (최소 8자, 영문+숫자+특수문자)
- 비밀번호 일치 확인
- 파트 선택 (기획/디자인/퍼블리싱/개발)

### 로그인 (login.html)
- 이메일/비밀번호 입력
- 로그인 실패 시 에러 메시지
- 로그인 성공 시 대시보드로 이동

### 프로필 관리 (profile.html)
- 본인 정보 조회
- 이름, 파트 수정 가능
- 이메일, 등급은 읽기 전용
- 비밀번호 변경 링크

### 비밀번호 변경 (password.html)
- 현재 비밀번호 확인
- 새 비밀번호 정책 검증
- 새 비밀번호 일치 확인

### 사용자 목록 (users.html)
- 전체 사용자 목록 테이블
- 검색 (이름, 이메일)
- 필터 (파트, 등급, 활성 상태)
- 페이지네이션

### 사용자 상세 (user-detail.html)
- 멤버 정보 수정 (파트, 등급)
- 활성/비활성 토글
- 비활성화 확인 다이얼로그

## 스타일 특징

- 다크모드 디자인
- 반응형 레이아웃
- 일관된 컬러 시스템
- 폼 검증 피드백
- 에러/성공 메시지 표시

## 주의사항

- 모든 링크는 상대 경로 사용
- 사이드바는 각 파일에 직접 포함 (CORS 방지)
- JavaScript는 시뮬레이션 수준 (실제 API 호출 없음)
- 비밀번호 정책: 최소 8자, 영문+숫자+특수문자 포함
- 권한 체크는 실제 구현에서 백엔드에서 처리
