# 프로젝트 관리 기획서

## 1. 개요

### 1.1 목적

웹에이전시의 신한카드 운영 및 고도화 업무를 체계적으로 관리하기 위한 프로젝트 관리 기능을 제공합니다.

### 1.2 배경

- 여러 프로젝트를 동시에 진행하면서 프로젝트별 정보를 관리할 필요성 증대
- 프로젝트 생성부터 완료까지의 전체 라이프사이클 관리 필요
- PM, PL, PA, 개발자 등 여러 역할의 사용자가 프로젝트 정보를 공유하고 협업

### 1.3 범위

**포함**
- 프로젝트 CRUD (생성, 조회, 수정, 삭제)
- 프로젝트 목록 조회 (검색, 필터링, 정렬)
- 프로젝트 상태 관리 (진행중, 완료, 중단)
- 프로젝트 기본 정보 관리 (명칭, 설명, 기간)

**제외 (향후 기능)**
- 프로젝트 멤버 관리 → 2단계 (리소스 관리)
- 프로젝트 권한 관리 → 3단계 (권한 시스템)
- 프로젝트 통계/리포트 → 4단계 (리포트)

## 4. 사용자 스토리

### US-01: 프로젝트 생성

**As a** PM  
**I want** 새 프로젝트를 생성하고  
**So that** 업무를 시작할 수 있다

**인수 조건 (Acceptance Criteria)**
- [ ] 프로젝트명은 필수 입력 (2-100자)
- [ ] 설명은 선택 입력 (최대 1000자)
- [ ] 시작일/종료일 설정 가능 (선택)
- [ ] 종료일은 시작일 이후여야 함
- [ ] 중복된 프로젝트명은 생성 불가 (에러 메시지 표시)
- [ ] 생성 성공 시 프로젝트 상세 화면으로 이동
- [ ] 생성 시 상태는 자동으로 "진행중"으로 설정

### US-02: 프로젝트 목록 조회

**As a** PM  
**I want** 모든 프로젝트 목록을 한눈에 보고  
**So that** 전체 프로젝트 현황을 파악할 수 있다

**인수 조건**
- [ ] 모든 활성 프로젝트가 카드 형태로 표시됨
- [ ] 각 카드에는 프로젝트명, 설명(요약), 기간, 상태가 표시됨
- [ ] 최신 생성순으로 정렬됨
- [ ] 카드 클릭 시 상세 화면으로 이동
- [ ] 프로젝트가 없을 경우 안내 메시지 표시

### US-03: 프로젝트 검색

**As a** PM  
**I want** 프로젝트명으로 검색하고  
**So that** 특정 프로젝트를 빠르게 찾을 수 있다

**인수 조건**
- [ ] 검색 입력란에서 프로젝트명 검색 가능
- [ ] 부분 일치 검색 지원 (contains)
- [ ] 검색 결과가 실시간으로 표시됨
- [ ] 검색 결과가 없을 경우 안내 메시지 표시

### US-04: 프로젝트 필터링

**As a** PM  
**I want** 상태별로 프로젝트를 필터링하고  
**So that** 진행중/완료/중단된 프로젝트를 구분해서 볼 수 있다

**인수 조건**
- [ ] 상태 선택 드롭다운 제공 (전체/진행중/완료/중단)
- [ ] 선택한 상태의 프로젝트만 표시됨
- [ ] 검색과 함께 사용 가능

### US-05: 프로젝트 상세 조회

**As a** PM  
**I want** 프로젝트의 모든 정보를 확인하고  
**So that** 프로젝트 현황을 파악할 수 있다

**인수 조건**
- [ ] 프로젝트명, 설명, 시작일, 종료일, 상태가 표시됨
- [ ] 생성일시, 수정일시가 표시됨
- [ ] [수정] 버튼으로 수정 화면 이동
- [ ] [삭제] 버튼으로 삭제 가능 (확인 다이얼로그)
- [ ] [← 목록으로] 버튼으로 목록 화면 이동

### US-06: 프로젝트 수정

**As a** PM  
**I want** 프로젝트 정보를 수정하고  
**So that** 변경된 정보를 반영할 수 있다

**인수 조건**
- [ ] 프로젝트명, 설명, 시작일, 종료일, 상태 수정 가능
- [ ] 프로젝트명 중복 검사 (현재 프로젝트 제외)
- [ ] 종료일 > 시작일 검증
- [ ] 수정 성공 시 상세 화면으로 이동
- [ ] 취소 시 변경사항 무시

### US-07: 프로젝트 삭제

**As a** PM  
**I want** 프로젝트를 삭제하고  
**So that** 불필요한 프로젝트를 제거할 수 있다

**인수 조건**
- [ ] 삭제 전 확인 다이얼로그 표시
- [ ] "정말 삭제하시겠습니까?" 메시지
- [ ] Soft delete 방식 (is_active = false)
- [ ] 삭제 성공 시 목록 화면으로 이동
- [ ] 삭제된 프로젝트는 목록에 표시되지 않음

### 5.3 화면 상세

#### SCR-01: 프로젝트 목록

**레이아웃**
- 헤더: "프로젝트 목록" 제목, [+ 새 프로젝트] 버튼 (우측 상단)
- 필터 영역: 검색 입력란, 상태 드롭다운
- 목록 영역: 프로젝트 카드 그리드

**프로젝트 카드**
- 프로젝트명 (링크)
- 설명 (요약, 100자)
- 기간 (YYYY-MM-DD ~ YYYY-MM-DD)
- 상태 뱃지 (진행중: 파란색, 완료: 녹색, 중단: 회색)

**검색/필터**
- 검색 입력란: placeholder "프로젝트명 검색"
- 상태 드롭다운: 전체 / 진행중 / 완료 / 중단

**빈 상태**
- 메시지: "프로젝트가 없습니다"
- [+ 새 프로젝트] 버튼

#### SCR-02: 프로젝트 생성

**폼 필드**
- 프로젝트명 * (텍스트 입력, 2-100자)
- 설명 (텍스트 영역, 최대 1000자)
- 시작일 (날짜 선택)
- 종료일 (날짜 선택)

**버튼**
- [생성] (파란색, 주 버튼)
- [취소] (회색, 보조 버튼)

**유효성 검증**
- 프로젝트명 필수
- 종료일 >= 시작일
- 중복 프로젝트명 에러

**에러 메시지**
- "프로젝트명을 입력해주세요"
- "이미 존재하는 프로젝트명입니다"
- "종료일은 시작일 이후여야 합니다"

#### SCR-03: 프로젝트 상세

**정보 표시**
- 프로젝트명 (헤더)
- 설명
- 시작일 ~ 종료일
- 상태
- 생성일시
- 수정일시

**버튼**
- [수정] (회색, 우측 상단)
- [삭제] (빨간색, 우측 상단)
- [← 목록으로] (하단)

**삭제 확인 다이얼로그**
- 메시지: "정말 삭제하시겠습니까?"
- [확인] / [취소]

#### SCR-04: 프로젝트 수정

프로젝트 생성(SCR-02)과 동일한 폼 구조, 단 기존 값이 미리 채워짐

**버튼**
- [수정] (파란색)
- [취소] (회색)

## 6. API 명세

### 6.1 API 목록

| Method | Path | 설명 |
|--------|------|------|
| GET | /api/projects | 프로젝트 목록 조회 |

### 6.2 API 상세

#### GET /api/projects

**Query Parameters**
- `search`: string (선택) - 프로젝트명 검색 (부분 일치)
- `status`: string (선택) - 상태 필터 (ACTIVE/COMPLETED/SUSPENDED)
- `sort`: string (선택) - 정렬 기준 (createdAt/updatedAt)
- `order`: string (선택) - 정렬 순서 (asc/desc), 기본값 desc

**Response 200**
```json
[
  {
    "id": "1",
    "projectName": "신한카드 PMS",
    "description": "프로젝트 관리 시스템 개발",
    "startDate": "2024-01-01",
    "endDate": "2024-12-31",
    "status": "ACTIVE",
    "createdAt": "2024-01-01T00:00:00Z",
    "updatedAt": "2024-01-02T00:00:00Z"
  }
]
```

#### GET /api/projects/:id

**Path Parameters**
- `id`: string (필수) - 프로젝트 ID

**Response 200**
```json
{
  "id": "1",
  "projectName": "신한카드 PMS",
  "description": "프로젝트 관리 시스템 개발",
```

## 7. 데이터 모델 (개념적)

### 프로젝트 (Project)

**주요 속성**
- id: 고유 식별자
- projectName: 프로젝트명 (필수, 유니크)
- description: 설명 (선택)
- startDate: 시작일 (선택)
- endDate: 종료일 (선택)
- status: 상태 (ACTIVE/COMPLETED/SUSPENDED)
- isActive: 활성 여부 (soft delete용)
- createdBy: 생성자 ID
- createdAt: 생성일시
- updatedBy: 수정자 ID
- updatedAt: 수정일시

**관계**
- (향후) Project 1:N ProjectMember (프로젝트 멤버)
- (향후) Project 1:N Task (작업)
- (향후) Project 1:N Issue (이슈)

## 8. 초기 데이터 (도메인 코드)

## 9. 예외 처리

| 케이스 | HTTP 상태 | 에러 메시지 | 처리 방법 |
|--------|-----------|-------------|-----------|
| 프로젝트명 미입력 | 400 | 프로젝트명을 입력해주세요 | 필수 입력 검증 |
| 프로젝트명 길이 초과 | 400 | 프로젝트명은 100자 이내여야 합니다 | 길이 검증 |
| 중복 프로젝트명 | 400 | 이미 존재하는 프로젝트명입니다 | 유니크 제약 검증 |
| 종료일 < 시작일 | 400 | 종료일은 시작일 이후여야 합니다 | 날짜 검증 |
| 존재하지 않는 프로젝트 | 404 | 프로젝트를 찾을 수 없습니다 | ID 존재 여부 검증 |
| 서버 오류 | 500 | 서버 오류가 발생했습니다 | 예외 로깅 및 일반 메시지 |

## 10. 제약사항

- 삭제된 프로젝트는 복구 불가 (soft delete, is_active = false)
- 프로젝트명은 대소문자 구분 (향후 요구사항에 따라 변경 가능)
- 생성 시 status는 자동으로 ACTIVE로 설정
- 시작일/종료일은 선택 사항 (기간이 정해지지 않은 프로젝트 허용)

## 11. 향후 확장

- 프로젝트 멤버 관리 (역할: PM, PL, PA, 개발자 등)
- 프로젝트 권한 관리 (생성자/PM만 수정/삭제 가능)
- 프로젝트 템플릿 (자주 사용하는 프로젝트 구조 저장)
- 프로젝트 복제 기능
- 프로젝트 아카이브 (완료된 프로젝트 별도 보관)
- 프로젝트 통계 (진행률, 완료율 등)
